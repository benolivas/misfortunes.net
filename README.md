# misfortunes.net

A fortune cookie site with an attitude. Displays random fortunes from a large curated list, with a custom web-based editor for managing content.

## Live site

[misfortunes.net](https://misfortunes.net)

## Structure

```
misfortunes.net/
├── index.html          # Main page
├── main.js             # Fortune logic, cookie/localStorage handling
├── fortuneArray.js     # All fortune strings
├── style.css           # Site styling
├── .htaccess           # CORS headers, cache rules, security
├── images/
│   └── fortune-cookie.png
└── editor/
    ├── index.html      # Fortune editor UI
    ├── auth.php        # Login handler (password-free, reads config.php)
    ├── push.php        # Writes fortuneArray.js to server
    └── config.php      # Password config — NOT in repo, manual deploy only
```

## Editor

A password-protected web editor lives at `/editor/`. Features:

- Fetches live `fortuneArray.js` on login — always in sync
- Live preview of each fortune on the cookie slip at correct size and rotation
- Fit indicator — flags fortunes that are too long
- Add, edit, delete fortunes
- Push to live — writes directly to `fortuneArray.js` on the server

## Fortune display logic

- First visit shows the newest fortune (last in array)
- Subsequent visits pick randomly from unseen fortunes — no repeats
- Seen list stored in `localStorage`, expires after 7 days
- After all fortunes seen, shows a final message then resets

## Local setup

```bash
git clone https://github.com/benolivas/misfortunes.net.git
cd misfortunes.net
```

Create `editor/config.php` manually (not in repo):

```php
<?php
define('EDITOR_PASSWORD', 'your_password_here');
define('TOKEN_EXPIRY', 28800);
```

## Hosting

Hosted on Dreamhost (Apache, PHP 8.3). 

## Not in this repo

- `editor/config.php` — contains editor password, lives on server and local machines only
- `editor/.token` — session token generated by server
